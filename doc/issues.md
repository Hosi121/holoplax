# Issues (high-value next steps)

## 1) 承認付き自動化フローを仕上げる
- 背景: 高スコア自動分解は承認待ちタグを付けるだけで、ユーザー操作で実行/却下できない。
- スコープ: 承認API（approve/skip）を追加し、バックログUIで「承認して分解」「却下」のボタンを表示。承認時は `lib/automation.ts` と同等ロジックで子タスク生成、却下時は承認待ちタグを外す。
- 受入れ: 1) 承認待ちカードにボタンが出る 2) 承認で子タスク生成＋親に分解済タグ 3) 却下でタグ削除のみ。

## 2) インボックス最小実装（メモ取り込み）
- 背景: 手入力のみで、要件の「複数インボックスから自動取込」が未着手。導入障壁が高い。
- スコープ: `/api/intake/memo` でメモをPOSTするとバックログにタスク生成し、重複判定（タイトル類似）を返す。UIに「メモ貼り付け→分析中→重複候補〇/×」の最小フローを追加。
- 受入れ: 1) API経由でタスク作成 2) 類似タスク候補がレスポンスに含まれる 3) UIで候補をマージor新規作成を選択できる。

## 3) キャパ/依存のガードレール
- 背景: スプリント移動でキャパ超過・依存未完でも通過でき、計画精度が落ちる。
- スコープ: バックログ→スプリント移動時に残容量チェックと依存未完警告を行う。超過時はエラー、依存未完は確認ダイアログ。ダッシュボードに「キャパ使用率」と「未解決依存数」を表示。
- 受入れ: 1) キャパ超過移動が阻止される 2) 未完依存は警告付きで操作 3) ダッシュボードにメトリクスが反映される。
