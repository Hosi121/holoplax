# Issues (high-value next steps, unresolved)

## 1) インボックス最小実装（メモ取り込み）
- 背景: 手入力のみで、要件の「複数インボックスから自動取込」が未着手。導入障壁が高い。
- スコープ: `/api/intake/memo` でメモをPOSTするとバックログにタスク生成し、重複判定（タイトル類似）を返す。UIに「メモ貼り付け→分析中→重複候補〇/×」の最小フローを追加。
- 受入れ: 1) API経由でタスク作成 2) 類似タスク候補がレスポンスに含まれる 3) UIで候補をマージor新規作成を選択できる。

## 2) AI提案を「適用」できるアクション化
- 背景: AI提案ログは閲覧のみで、タスクへの反映は手作業。AI価値を定着させにくい。
- スコープ: バックログ/設定の提案カードに「適用」ボタンを付け、分解・スコア更新・メモ追記をワンクリックで反映。適用履歴をAIログに紐付け。
- 受入れ: 1) 提案カードに適用ボタン 2) 押下でタスクが更新/分解される 3) 適用がログに記録される。

## 3) ダッシュボードを実データのみで表示（空データCTA強化）
- 背景: ベロシティ/バーンダウンがダミー系列を表示し、実績との乖離で信用を損なう。
- スコープ: フォールバック系列を撤廃し、データゼロ時はCTA（スプリント開始/タスク追加/インテグレーション設定）を表示。実データから平均/最高値を計算。
- 受入れ: 1) データがない時にグラフは空＋明確なCTA 2) 値はDBの実データのみで算出 3) 見た目の崩れがない。

## 3) キャパ/依存のガードレール
- 背景: スプリント移動でキャパ超過・依存未完でも通過でき、計画精度が落ちる。
- スコープ: バックログ→スプリント移動時に残容量チェックと依存未完警告を行う。超過時はエラー、依存未完は確認ダイアログ。ダッシュボードに「キャパ使用率」と「未解決依存数」を表示。
- 受入れ: 1) キャパ超過移動が阻止される 2) 未完依存は警告付きで操作 3) ダッシュボードにメトリクスが反映される。
